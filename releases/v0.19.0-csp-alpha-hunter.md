# v0.19.0 — CSP Alpha Hunter

**Date:** 2026-05-02

## Overview

Autonomous CSP (Cash Secured Put) discovery engine. Scans the market every 4 hours, identifies the most "juicy" puts using $100K in available capital, tracks changes between scans, and sends delta-aware reports with AI risk analysis.

## New Files

### Core Engine
- `src/lib/options/csp-scanner.ts` — Core scanner: fetches options chains across 40+ tickers, filters by DTE (7-21), delta (0.15-0.30), AROC (10%+), enriches with technicals and earnings
- `src/lib/options/csp-delta.ts` — Delta tracker: compares scans, detects new entries, premium changes, dropped candidates, lost support levels. Persists to Supabase.
- `src/lib/options/csp-analyst.ts` — Claude risk analyst: sends top 15 candidates to Claude with full context for portfolio-level risk assessment

### API Routes
- `src/app/api/cron/csp-hunter/route.ts` — Cron endpoint (CRON_SECRET auth), runs full scan → delta → analysis → email pipeline
- `src/app/api/csp-hunter/route.ts` — GET recent scans, POST manual trigger (auth-protected)

### Notifications

### UI

## Key Types
- `CSPHunterCandidate` — Full candidate with Greeks, capital metrics, technicals, earnings, juiciness score
- `CSPScanResult` — Scan output with all candidates, errors, metadata
- `ScanDelta` — Delta between scans (new, premium_up, premium_down, dropped, support_lost)
- `CSPScanConfig` — Configurable: capital, DTE range, delta range, AROC threshold, ticker universe

## Juiciness Score (0-100)
- AROC (0-30 pts) — annualized return on capital
- Delta optimality (0-15 pts) — closer to 0.20 is ideal
- Distance OTM (0-10 pts) — more OTM = safer
- Technical support (0-10 pts) — strike near EMA support
- Technical score (0-10 pts) — composite from full analysis
- Liquidity (0-10 pts) — OI and volume
- IV premium (0-10 pts) — higher IV = more premium
- Earnings penalty (-15 pts) — risk factor for earnings within DTE
- RSI adjustments (±5 pts) — overbought/neutral
