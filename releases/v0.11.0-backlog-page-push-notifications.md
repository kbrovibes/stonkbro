# v0.11.0 — Backlog Status Page, Push Notifications, Auto-Update Hook

## Summary
Static backlog status page for GitHub Pages, Web Push notification support in the PWA, and a Claude Code hook that auto-regenerates the page and sends notifications when BACKLOG.md changes.

## Dependencies added
- `web-push` — server-side Web Push protocol
- `@types/web-push` — TypeScript declarations

## New environment variables
| Variable | Scope | Purpose |
|---|---|---|
| `NEXT_PUBLIC_VAPID_PUBLIC_KEY` | Public | VAPID public key for Web Push subscription |
| `VAPID_PRIVATE_KEY` | Server | VAPID private key for signing push messages |

## Notes
- VAPID keys generated via `npx web-push generate-vapid-keys` and stored in `.env.local`. Must also be added to Vercel env vars for production.
- The push notification send endpoint uses CRON_SECRET for auth (same as the cron job), so backlog hook needs that env var set locally.
- Push subscriptions auto-cleaned when endpoints return 410/404 (expired).
