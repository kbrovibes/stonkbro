# v0.18.1 — Sparkline Caching

## Summary
Implements server-side caching for ticker sparklines to significantly improve home page and watchlist performance.

## Files changed
| File | Change |
|---|---|
| `supabase/migrations/20260427_market_sparklines.sql` | New table `market_sparklines` for caching |
| `src/app/api/cron/sparklines/route.ts` | New background job to pre-warm the cache |
| `src/app/api/sparklines/route.ts` | Refactored to use cache with 2-hour TTL |
| `src/app/(app)/page.tsx` | Minor lint fix for variable declaration |

## Notes
- Sparkline data (5-day close prices) is now stored as JSONB in Supabase.
- The cron job scans all watchlists and active positions to ensure the cache is warm for all relevant tickers.
- The API route handles cache misses by fetching live data and updating the cache asynchronously.
- Performance: Reduces parallel history fetches on page load, moving the heavy lifting to a background process.
