# v0.22.0 — Portfolio Manager

## Summary
Daily AI-driven research, BUY/SELL ratings, and a $100K reallocation plan for every stock in the user's connected SnapTrade portfolio. Standalone page reachable from More → Portfolio. Runs on cron at market open and rides along the existing close cron; manual re-run button supported with concurrency guard.

## Dependencies added
- None (uses existing `@anthropic-ai/sdk`, `@google/generative-ai`, SnapTrade SDK, Supabase admin, Tradier/Yahoo market data, Yahoo Finance unofficial news endpoint)

## Notes
- **AI cost**: batched single call covering all tickers + the allocation plan. For a 25-ticker portfolio that's ~15K input + ~5K output tokens — roughly $0.05–0.15 per run on Claude Sonnet 4.5, or pennies on Gemini Flash. Provider/model honored from `user_settings.preferred_ai_provider` / `preferred_ai_model` with the same auto-fallback path used by csp-hunter.
- **Holdings cache**: keyed by UTC `date` so the second scan of the day (e.g. close ride-along after the open cron) skips the SnapTrade call. Free cash is recomputed each time since balances drift intraday.
- **Yahoo Finance news**: unofficial endpoint — no API key, but fragile. Failures are silent and degrade to "(no headlines)" in the prompt; analysis still runs.
- **Cron count**: bumped from 14 to 15 entries. Hobby plan limit is the next risk — if it trips, drop one of the 4 recommendations entries or one of the 3 csp-hunter entries.
- **Concurrency**: `/scan` POST returns 409 if another scan started in the last 10 minutes is still `running`, preventing duplicate AI charges.
- **Schema validation**: `analyst.ts` tolerates messy AI output (case-insensitive ratings, missing fields, prose around the JSON block) and falls back to `HOLD` rather than failing the whole scan.
- **No BottomNav change** by user request — page lives in More only.
- **Page caching**: `dynamic = "force-dynamic"` on the page; the heavy lifting happens server-side, but page reads only the latest persisted row (no AI on page load).
