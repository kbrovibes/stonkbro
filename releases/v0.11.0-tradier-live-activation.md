# v0.11.0 — Tradier Live Data Activation

**Date:** 2026-04-26

## What changed

Centralized all Tradier API access into a single client module with proper rate limit handling and environment-aware URL switching.

### New: `src/lib/market/tradier-client.ts`
- `getTradierBase()` / `getTradierBaseV1()` — returns sandbox or production URL based on `TRADIER_ENV`
- `getTradierHeaders()` — centralized auth header generation
- `tradierFetch(path, options)` — smart fetch wrapper:
  - Reads `X-Ratelimit-Allowed`, `X-Ratelimit-Used`, `X-Ratelimit-Available`, `X-Ratelimit-Expiry` from every response
  - Exponential backoff on 429 (1s → 2s → 4s) with configurable max retries
  - Fast-fail on 401 (invalid token)
  - Network error retry with backoff
- `getRateLimitState()` — exposes current rate limit quota for monitoring

### Updated modules
- **tradier.ts** — replaced raw `fetch` + local `getHeaders()` / `TRADIER_BASE` with `tradierFetch`
- **history.ts** — replaced hardcoded `https://sandbox.tradier.com/v1` with `tradierFetch`
- **earnings.ts** — replaced local `TRADIER_BASE` + `getHeaders()` with centralized imports
- **api/tradier-usage** — uses centralized client, response now includes `rateLimit` object

### Environment support
- `TRADIER_ENV=production` → `https://api.tradier.com` (all endpoints)
- Default (unset or `sandbox`) → `https://sandbox.tradier.com`
