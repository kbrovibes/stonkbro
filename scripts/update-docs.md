# Documentation Update Instructions

When a feature change is committed to stonkbro (new pages, API routes, database changes, or feature additions/removals), update these files:

## README.md
- Update the Features section if a feature was added, removed, or significantly changed
- Update the Tech Stack table if dependencies changed
- Update the API Routes table if endpoints were added/changed
- Update the Roadmap if backlog items were completed or added
- Update the Version History table if a new version was released
- Do NOT rewrite the entire README — only update the affected sections

## docs/index.html
- Update the feature cards grid if features changed
- Update the API table if endpoints changed
- Update the tech stack cards if dependencies changed
- Update the version timeline if a new version was released
- Update the roadmap cards if backlog changed
- Update the hero badge version number if version bumped
- Do NOT rewrite the entire file — only update the affected sections

## What NOT to update
- Do not update docs for bug fixes, refactors, or style changes
- Do not update docs for changes that don't affect user-facing features
- Do not create a changelog entry — that's handled separately
